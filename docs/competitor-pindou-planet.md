## 1. 产品架构

五 Tab：首页（关注/笔记/图纸市场）· 赚豆子（任务 + 商城）· 做图纸（创作中心）·
消息 · 我的。本质是**以「豆子」虚拟货币为血液的 UGC 社区电商**，工具退居第三 Tab。

## 2. 变现结构（按重要性）

| 变现点 | 形态 | 观察到的定价 | 备注 |
|---|---|---|---|
| **保存图纸（计费时刻）** | 豆子按次扣费：AI 生成/预览免费（看广告，3 次免广告），**保存那一刻扣 1 豆**；注册赠 1 豆（免费尝一次） | **1 豆 = ¥2.8/张**（订单页实证） | 付费墙摆在「用户已喜欢结果、要带走」的瞬间，付费意愿最高点；服务器兑付成本≈0，毛利≈100% |
| **会员** | 订阅：无限生成·无限预览·无广告·免费保存 | 未知 | 把按次付费的摩擦打包买断，标准 per-use→subscription 路径 |
| **按图配豆** | 商城 SKU「根据图纸所需豆子数量发货（无需购买全套221）」 | **¥19.9 起** | 与 BeadFable 套装假设同构；国内 ¥19.9 → 海外 $19.9 即出海溢价空间 |
| 全套色号/标品 | 商城 | ¥9.9 级 | 标品，拼价格 |
| 图纸定制 | 人工手绘服务 | 未知（高客单） | 服务型长尾 |
| UGC 图纸售卖 | 免费/广告解锁/付费 | ¥0.5/张 量级 | 图纸本身不值钱，验证 PRD「图纸是引流品」判断 |

### 豆子经济的核心算术（第三批截图实证）
- 用户端 1 豆 = ¥2.8 真金白银；平台端 1 豆兑付成本 ≈ 0（一次保存/生成的边际成本近零）。
- 任务返豆（小红书好评 +5 豆）= 用户感知 **¥14 的报酬**、平台实际成本 **≈ 0**。
- 即：**用零成本内部货币按 ¥14/条的感知价格采购真实买家秀与社媒内容**，
  市场费用全部豆子化。「兑换码」是同一货币的 KOL 投放通道。

## 3. 增长引擎（全部众包）

- 小红书/淘宝**好评返豆**（+5 豆）：用虚拟货币收买真实买家秀，养自己店铺评价。
- 发抖音/小红书/快手提及小程序 +1~10 豆；邀请注册 +3 豆；精选图纸投稿 +3 豆。
- **导出图纸自带水印**：「打开拼豆星球，搜索【图纸编号】获得超清原图」——
  每张被分享/打印的图纸都是带回流入口的广告。图纸编号 = 搜索码 = 拉新链路。
- 作品详情页红字「获得方式：未发布广场」催发布 → 供给侧靠 UGC，官方饥渴。

## 4. 核心链路细节（第二批截图）

- **AI 风格化**：8 风格（大头Q版/小图Q版/迪士尼/原图/高度还原/Chibi/猫/车）+
  自定义文本要求（如「衣服改成红色、背景更简洁」）。
  ⚠️ 风格直接命名「迪士尼」——海外不可复制，BeadFable 用 Toon/Chibi/Big-head 等通用词。
- **作品详情**：大小/色号品牌(mard)/色号数量/浏览/使用 元数据规范；开始拼豆按钮。
- **编辑模式**：完整像素编辑器（画笔/橡皮/油漆桶/取色/选区/镜像/撤销重做），
  底部色板实时显示每色号用豆数。
- **拼豆模式**：逐格引导拼豆的互动模式（= 拼豆「游戏」的实际形态，留存功能）。
- **导出配置**：色号开关/统计/镜像（熨烫翻面用）/网格深浅/行列坐标/底色/辅助线，
  服务端渲染（「根据选项调用接口生成图片」）。
- **成品图纸**：每格印色号缩写、四边行列坐标、底部图例条含每色用量、
  作者署名、色号数/豆子数合计、回流水印。

## 4b. 用户中心与治理（第三批截图）

- 「我的」信息架构最小集合：我的笔记/我的图纸/收藏点赞/我的订单/已购图纸/
  豆子记录/编辑信息/商务合作/联系客服/兑换码；社交四件套（关注/粉丝/获赞/收藏）。
- 会员 banner 常驻个人页 C 位：「立即开通会员无限生成：无限预览·无广告·免费保存图纸」。
- **所有 UGC 图纸（含用户自己生成的免费图）都挂「待审核」**：人工审核队列既是
  内容治理也是 IP 挡箭牌，但是持续人力成本——再次确认小团队不要碰 UGC 广场。
- 「认证身份」（如绘画师徽章）：创作者供给侧分层运营。

## 4c. 私域与留存（第四批截图，拆解收尾）

- 「联系客服」= **管理员个人微信二维码 + 用户微信交流群二维码**，不是工单系统。
  用户资产沉淀到私域：复购提醒、新品发布、售后、晒单种草全在群内完成，
  不受平台规则约束（「关注公众号防失联」同属此线）。实物套装生意的复购引擎。
- 消息页：极简通知中心（点赞/评论/粉丝 + 系统通知）。
- **BeadFable 海外等价物**：个人微信+群 ≈ 邮件列表 + Discord/FB Group。
  V1 预售按钮收的邮箱即私域第一块砖；V1.1 加邮件订阅入口；
  晒单氛围先寄生 Reddit r/beadsprites 与 Pinterest，不自建社群（V2 再议）。

## 7. 商业闭环全景（四批拆解总结）

```
广告 + 按次保存费（¥2.8/张）养 AI 工具
  → 工具产出图纸（供给侧 UGC 化，全量审核）
    → 图纸带出「按图配豆」实物订单（¥19.9 起，收银台）
      → 豆子任务驱动用户去社媒拉新 + 好评（零成本市场预算）
        → 私域（个人微信 + 群）承接复购
```

BeadFable 对位：V1 已建第 2、3 环（图纸+BOM+套装预售）；V1.1 补第 1 环
（AI Q 版化，计费时刻在下载）；第 4 环换海外合规形态（分享得折扣）后置；
第 5 环从预售邮箱起步。

## 5. 它的软肋（= BeadFable 的窗口）

1. **默认参数坑新手**：实测照片直转 104×104 / 221 色全开 → 56 色、10,816 颗、
   效果糟。复杂度全丢给用户。BeadFable 反着做：强预设（29×29、≤12 色）、
   Q 版单一叙事、出来就是拼得完的东西。
2. **IP 风险内嵌**：付费风格叫「迪士尼」、UGC 公开售卖蜡笔小新图纸。国内灰色，
   海外致命。BeadFable 红线见 PRD §8。
3. **锁死在微信小程序 + 漫漫(mard)色号 + 中文世界**：出不了海的三道墙。
4. **重运营**：豆子经济、任务审核、UGC 审核都是人力黑洞；单人/小团队不可复制，
   也不必复制。
